关于其他继承方法

1. 简单原型链

实现

缺点

  1. 修改父类的实例域时，所有的对象都被修改
  2. 无法向父类构造函数传参
2. 借用构造函数

实现
解决了简单原型链方法的两个弊端

缺点

  1. 无法实现函数复用，每一个对象都持有一个function
3.原型式

实现
不需要创建自定义类型

缺点

  1. 父类原型中实例域共享
  2. 无法代码复用，创建子类麻烦
4.寄生式

实现
不需要创建自定义类型，是原型式的简单改进，穿上了函数马甲

缺点
不能实现函数复用(没有原型)
